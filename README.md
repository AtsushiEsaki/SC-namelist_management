# SC-namelist_management
同期の名簿管理、イベント登録、イベント出欠管理、近況報告等
# SC名簿・イベント管理システム企画書

## 1. 背景

定年退職を迎えた同期メンバーとのつながりを維持し、継続的なコミュニケーションを促進するため、同期専用のコミュニティサイトを構築する。

現在、同期会やゴルフコンペ等の案内はメールやLINE等で個別に実施しているため、

* 連絡先の管理が煩雑
* 出欠確認に時間がかかる
* 最新の住所・電話番号が分からない
* イベント履歴が残らない
* 幹事の負荷が高い

という課題が存在する。

本システムにより、同期コミュニティのデジタル基盤を構築し、幹事業務の効率化と会員同士の交流活性化を実現する。

---

# 2. 目的

## 会員管理

同期メンバーの連絡先を一元管理する。

## イベント管理

同期会・ゴルフコンペ・旅行・懇親会などのイベントをオンラインで管理する。

## 出欠管理

参加状況をリアルタイムで把握し、集計業務を自動化する。

## コミュニティ活性化

定期的なイベント案内や情報共有により同期の交流を促進する。

# 6. データベース設計

| ---------- | ---- |

| 項目          | 内容     |
| ----------- | ------ |
| id          | イベントID |

| event_date  | 開催日    |
| location    | 場所     |
| fee         | 会費     |
| description | 案内文    |

| 項目        | 内容       |
| --------- | -------- |

# 8. 将来構想

SCコミュニティプラットフォームとして発展

提供機能

* 同期会管理
* ゴルフコンペ管理
* 写真アルバム
* メッセージ機能
* 訃報・近況共有
* 会費徴収
* オンライン名簿

同期ネットワークを長期的に維持するためのデジタル基盤を目指す。

この企画であれば、まずは **GitHub Codespaces + Supabase + Vercel** で数日以内にMVPを構築でき、将来的には「同期会ポータル」「ゴルフコンペ管理」「会費決済」まで拡張可能です。特に定年後の同期コミュニティでは、名簿管理よりも「イベント運営の効率化」が利用価値の中心になるため、イベント作成・出欠管理・メール通知を優先実装するのが良いと思います。
